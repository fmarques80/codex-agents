# Playbook de Desenvolvimento de Feature

## Objetivo

Orientar a criação ou alteração de funcionalidades com segurança, clareza e baixo risco.

---

## Quando Usar

Usar quando o usuário solicitar:

- nova funcionalidade
- alteração de funcionalidade existente
- novo fluxo
- nova API
- nova tela
- nova integração
- mudança em regra de negócio

---

## Fluxo

1. Entender objetivo da feature
2. Verificar AGENTS.md local
3. Ler documentação relacionada
4. Identificar domínio afetado
5. Identificar agentes necessários
6. Analisar impacto técnico
7. Analisar impacto em segurança
8. Analisar impacto em testes
9. Implementar de forma incremental
10. Atualizar documentação quando necessário
11. Revisar código
12. Validar testes
13. Consolidar resposta final

---

## Agentes Possíveis

- Backend Engineer
- Frontend Engineer
- Database Engineer
- Mobile Capacitor Engineer
- Security Engineer
- QA Engineer
- Documentation Engineer
- Code Reviewer

---

## Saída Esperada

A resposta deve incluir:

- resumo da feature
- arquivos afetados
- riscos
- testes necessários
- documentação atualizada
- próximos passos
