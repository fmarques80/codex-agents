# Playbook de Revisão Arquitetural

## Objetivo

Orientar análise arquitetural de aplicações, módulos, features ou propostas técnicas.

---

## Quando Usar

Usar quando houver:

- mudança estrutural
- dúvida arquitetural
- proposta de migração
- avaliação de microserviços
- avaliação de Docker
- avaliação de Kubernetes
- avaliação de Terraform
- revisão de escalabilidade
- revisão de monorepo
- revisão de separação de domínios

---

## Fluxo

1. Entender objetivo da análise
2. Verificar AGENTS.md local
3. Ler documentação relevante
4. Identificar arquitetura atual
5. Identificar restrições reais
6. Identificar gargalos reais
7. Avaliar tradeoffs
8. Avaliar impacto operacional
9. Avaliar segurança
10. Avaliar custo
11. Avaliar alternativas mais simples
12. Recomendar evolução incremental
13. Documentar decisão quando necessário

---

## Agentes Possíveis

- CTO Agent
- Backend Engineer
- Frontend Engineer
- Database Engineer
- DevOps Engineer
- Cloud Engineer
- Security Engineer
- Performance Engineer
- Documentation Engineer
- Code Reviewer

---

## Saída Esperada

A resposta deve incluir:

- diagnóstico arquitetural
- tradeoffs
- riscos
- recomendação
- alternativa mais simples
- impacto operacional
- necessidade ou não de ADR
