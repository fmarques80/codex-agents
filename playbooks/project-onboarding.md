# Playbook de Onboarding de Projeto

## Objetivo

Orientar o CTO Agent e os especialistas na análise inicial de uma aplicação existente.

---

## Quando Usar

Usar quando:

- o projeto já existe
- a aplicação ainda não possui AGENTS.md local
- há pouca documentação
- será iniciado trabalho em código legado
- é necessário entender arquitetura antes de modificar

---

## Fluxo

1. Localizar raiz do projeto
2. Procurar AGENTS.md local
3. Ler README principal
4. Ler documentação em `/docs`
5. Analisar package.json
6. Identificar stack real
7. Identificar estrutura de diretórios
8. Identificar scripts importantes
9. Identificar infraestrutura
10. Identificar APIs e domínios principais
11. Identificar riscos
12. Propor ou criar AGENTS.md local
13. Propor documentação complementar

---

## Saída Esperada

O CTO Agent deve entregar:

- resumo da aplicação
- stack identificada
- estrutura principal
- riscos iniciais
- agentes recomendados
- documentação ausente
- próximos passos
